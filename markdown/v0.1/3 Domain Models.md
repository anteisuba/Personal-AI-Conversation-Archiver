## 3.1 Message

Represents a single message in a conversation.

export type MessageRole = 'system' | 'user' | 'assistant' | 'tool'  
  
export type Message = {  
  id: string  
  role: MessageRole  
  content: string  
  createdAt?: string  
}

### Notes

- `id` should be unique within the conversation.
    
- `createdAt` is optional because some sources do not expose per-message timestamps.
    
- `tool` is reserved for future compatibility.

## 3.2 NormalizedConversation

Unified conversation structure after parsing a source-specific payload.

export type ConversationSource =  
  | 'chatgpt'  
  | 'claude'  
  | 'gemini'  
  | 'cursor'  
  | 'manual'  
  
export type ConversationLanguage = 'zh' | 'ja' | 'en' | 'mixed'  
  
export type NormalizedConversation = {  
  id: string  
  source: ConversationSource  
  externalId?: string  
  title: string  
  createdAt: string  
  updatedAt?: string  
  messages: Message[]  
  metadata: Record<string, unknown>  
  
  language?: ConversationLanguage  
  messageCount?: number  
  tokenEstimate?: number  
  contentHash?: string  
}

### Field descriptions

- `id`: internal stable ID generated by the system
    
- `source`: source type of imported conversation
    
- `externalId`: original source identifier if available
    
- `title`: normalized title, generated from source title or fallback logic
    
- `createdAt`: source-level creation time or import-derived fallback
    
- `updatedAt`: source-level update time if available
    
- `messages`: ordered conversation messages
    
- `metadata`: source-specific extra fields
    
- `language`: locally detected language
    
- `messageCount`: number of messages
    
- `tokenEstimate`: locally estimated token size
    
- `contentHash`: hash of normalized content for change detection and future dedupe

## 3.3 ConversationMetaAnalysis

Structured output from LLM call #1.

export type ConversationCategory =  
  | 'programming'  
  | 'project'  
  | 'career'  
  | 'language'  
  | 'ai-tools'  
  | 'life'  
  
export type ConversationMetaAnalysis = {  
  conversationId: string  
  category: ConversationCategory  
  tags: string[]  
  project: string | null  
  confidence: number  
  analyzedAt: string  
}

### Notes

- Categories are intentionally limited in v0.1.
    
- Fine-grained classification should be represented via `tags`.
    
- `confidence` is used for later inspection and manual review workflows.

## 3.4 RefinedNote

Structured output from LLM call #2.

export type RefinedNote = {  
  conversationId: string  
  title: string  
  summary: string  
  keyPoints: string[]  
  actionItems: string[]  
  reusablePrompts: string[]  
  rewrittenNote: string  
  refinedAt: string  
}

### Notes

- `rewrittenNote` is the main body to be exported into Markdown.
    
- `summary` should be concise and high-signal.
    
- `keyPoints` should be suitable for list rendering.
    
- `actionItems` may be empty.
    
- `reusablePrompts` may be empty.

## 3.6 Job

Represents a batch-level execution record.

export type JobType = 'import' | 'process' | 'publish' | 'run'  
  
export type JobStatus = 'pending' | 'running' | 'completed' | 'failed' | 'partial'  
  
export type Job = {  
  id: string  
  type: JobType  
  sourcePath?: string  
  status: JobStatus  
  totalItems: number  
  processedItems: number  
  failedItems: number  
  startedAt: string  
  finishedAt?: string  
  lastError?: string  
}

---

## 3.7 JobItem

Represents item-level tracking within a job.

export type PipelineStage =  
  | 'imported'  
  | 'normalized'  
  | 'enriched'  
  | 'analyzed'  
  | 'refined'  
  | 'published'  
  | 'failed'  
  
export type JobItemStatus = 'pending' | 'running' | 'completed' | 'failed'  
  
export type JobItem = {  
  jobId: string  
  conversationId: string  
  stage: PipelineStage  
  status: JobItemStatus  
  attemptCount: number  
  lastError?: string  
  updatedAt: string  
}

### Notes

- `JobItem` enables resume and retry.
    
- The same conversation may appear in multiple jobs over time.
