## Package Structure

### apps/cli

User-facing command line interface.

### packages/core

Core domain types, adapters, normalization logic, local enrichment, and pipeline orchestration.

### packages/ai

LLM providers, prompts, structured schemas, and AI tasks.

### packages/storage

File-based stores, SQLite repositories, and Markdown exporter.

## 6. Data Locations

### Repository

Stores source code only.

### App Data Directory

Recommended default:  
`~/.ai-archive/data`

Contains:

- raw imported files
    
- normalized data
    
- processed data
    
- SQLite database
    

### Obsidian Vault

Configured separately by user.  
Recommended example:  
`~/Documents/Obsidian/AI-Vault`

## 7. Core Domain Models

### NormalizedConversation

Unified conversation structure after source parsing.

### ConversationMetaAnalysis

Structured metadata from the first LLM pass:

- category
    
- tags
    
- project
    
- confidence
    

### RefinedNote

Structured knowledge artifact from the second LLM pass:

- summary
    
- key points
    
- action items
    
- reusable prompts
    
- rewritten note
    

## 8. LLM Strategy

To keep v0.1 simple and cost-effective, the system uses only 2 LLM calls per conversation.

### Call #1: Analyze Meta

Output:

- category
    
- tags
    
- project
    
- confidence
    

### Call #2: Refine Note

Output:

- summary
    
- key points
    
- action items
    
- reusable prompts
    
- rewritten note
    

Language detection is handled locally and does not use an LLM.

## 9. Job and Resume Strategy

The system tracks both batch-level and item-level progress.

### jobs table

Tracks overall batch progress.

### job_items table

Tracks per-conversation stage, retry count, and failure info.

This enables:

- resuming after crash
    
- retrying failed items
    
- partial progress visibility
    

## 10. Export Strategy

Markdown notes are published into the Obsidian vault.

The vault contains knowledge-oriented notes, not full raw chat dumps.

Suggested routing:

- category-based folders
    
- optional project-based folders
    
- summary folders for future weekly/monthly views
    

## 11. Future Extensions

Planned after v0.1:

- dedupe
    
- weekly summary
    
- project knowledge aggregation
    
- browser extension
    
- Obsidian plugin
    
- semantic similarity clustering
